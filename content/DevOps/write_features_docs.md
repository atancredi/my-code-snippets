
# Brief Introduction
An introductory paragraph on the feature's purposes, like what the feature does or what problem it resolves.
# Deployment infos
This is intended more for "pitching" purposes, less for a technical doc.
Explain how the feature is going to be deployed and if possible give an ETA.
# How to test
This is important for technical docs: how can the feature be accessed by reviewers?
Include instructions for local testing if possible as well as instructions for integration or staging environments.
# Detailed infos
In technical documents is fundamental to explain in detail every decision made while developing the feature, with particular emphasis on any tradeoff.

Explain every step of the UX flow and what happens in the back-end as well.

Give details on the data used and produced by the new feature, as well as details on error cases and exception handling.
